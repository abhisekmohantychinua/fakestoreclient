FakeStoreClient Angular Application Documentation

Overview

This Angular application is designed to interact with the FakeStore API, which provides a simulated online store environment. It's important to note that the entire application is virtual and no actual database operations are being performed. The purpose of this application is for testing and development, allowing users to work with realistic e-commerce data without the need for a real backend. Before you start must read the Docs

Important Note

This application is designed as a virtual representation of an e-commerce store and is intended for testing and development purposes only. Please be aware of the following key points:

  • All operations within this application, including adding to cart, creating users, and any create, update, or delete actions, are entirely virtual.
  • No data is uploaded or stored in a real database. Each time you refresh or reload the website, you will receive the same results.
  • Some features, such as purchasing products or leaving ratings, have not been implemented due to the virtual nature of this application.

We hope you find this application helpful for testing and development purposes. If you have any questions or feedback, please feel free to reach out to us.

Application Structure

Components

  • Banner: Displays banner posters, commonly used for advertisements.
  • Back Arrow: Provides a navigational back arrow for improved user experience.
  • Navbar: The navigation bar that allows users to easily navigate through the application.
  • Product Card: Represents a product with its relevant details like title, category, and price.
  • Profile: Displays user profile information and options for account management.
  • Snackbar: Provides non-intrusive notifications to the user for actions or updates.
  • Suggestions: Offers product suggestions to enhance user engagement and exploration.

Services

  • AuthService: Responsible for managing user authentication and providing functions for login, logout, and user information retrieval.
  • ProductService: Handles interactions with the FakeStore API for product-related operations, such as retrieving products by category, fetching categories, and managing user carts.

Routing

  • Home: Displays the banner and product suggestions.
  • Login: Provides a login interface for users.
  • Product Details: Shows detailed information about a specific product.
  • Category: Displays a list of products belonging to a specific category.
  • Cart: Shows the user's shopping cart.
  • Documentation: Provides detailed documentation about the application.

Functionality

Products

  • Users can view a list of available products.
  • Clicking on a product will display detailed information.

Categories

  • Users can browse products by category.
  • Clicking on a category will filter the product list accordingly.

User Authentication

  • Users can log in to their accounts.
  • Upon successful login, the user's authentication token is stored locally.
